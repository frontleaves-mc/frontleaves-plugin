# INTERNAL KNOWLEDGE BASE

## OVERVIEW
`internal/` implements the business runtime: route → handler → logic → repository → entity, plus dual-protocol (HTTP + gRPC) entry points.

## STRUCTURE
```text
internal/
├── app/
│   ├── grpc/          # Inter-service gRPC client (yggleaf AuthClient)
│   ├── middleware/     # Gin middleware (LoginAuth, RequireRole)
│   ├── route/         # Route groups + middleware binding
│   └── startup/       # Infra init + startup node registration
├── handler/           # HTTP input/output boundary
├── logic/             # Business orchestration + SchedulerEngine
├── repository/        # DB access + Redis cache
│   └── cache/         # xCache.Cache Redis hash managers
├── entity/            # GORM models with snowflake gene binding
├── constant/          # Cache keys, env keys, gene numbers, event types
├── grpc/
│   ├── handler/       # gRPC service handlers (stream + unary)
│   ├── middleware/     # gRPC interceptors (PluginVerify)
│   ├── register/      # Service registration + engine init
│   └── gen/           # Auto-generated protobuf Go (buf output, DO NOT EDIT)
├── sse/               # Server-Sent Events (chat stream registry)
└── util/              # Shared utilities (markdown strip)
```

## WHERE TO LOOK
| Task | File/Dir | Why |
|---|---|---|
| Add route group | `app/route/route.go` + `route_*.go` | Route wiring entry is `NewRoute` |
| Add new handler | `handler/handler.go`, `handler/*.go` | `NewHandler[T]` auto-wires all logic deps |
| Add new logic module | `logic/*.go` | Logic takes context-injected db/rdb |
| Add new repository | `repository/*.go` | Repository returns `*xError.Error` |
| Add cache manager | `repository/cache/*.go` | Use `xCache.Cache` type alias + `RedisKey.Get()` |
| Add new table/entity | `entity/*.go` + `app/startup/startup_database.go` | Append to `migrateTables` in FK order |
| Add Snowflake gene | `constant/gene_number.go` | One gene per entity type |
| Add Redis cache key | `constant/cache.go` | Use `RedisKey.Get()` for prefix formatting |
| Add gRPC service | `grpc/handler/` + `grpc/register/register.go` | Register in `RegisterGRPCServices` |
| Add gRPC interceptor | `grpc/middleware/` | Must provide both Unary + Stream pair |
| Add SSE endpoint | `sse/chat_stream.go` | In-process client registry pattern |

## CONVENTIONS
- Keep handlers thin: validate/bind → call logic → map result/err to response.
- Logic is orchestration only; persistence/SQL belong in repository layer.
- Repositories expose `(data, *xError.Error)` style APIs, not raw `error`.
- gRPC handlers follow same layering: `grpcHandler` base → service sub-structs → logic calls.
- Use `xLog.WithName` by layer: `NamedCONT`/`NamedLOGC`/`NamedREPO`/`NamedINIT`/`NamedMIDE`/`NamedGRPC`.
- Use request context for downstream calls (`ctx.Request.Context()` or injected context).
- `bConst` alias for `internal/constant` package throughout internal code.

## ANTI-PATTERNS
- Do not access DB/Redis in handlers directly.
- Do not return HTTP JSON directly from logic/repository layers.
- Do not construct ad-hoc handlers bypassing `NewHandler[T]` or gRPC handler patterns.
- Do not place business constants inside handler/logic files; keep in `constant/`.
- Do not edit `grpc/gen/` — auto-generated by `buf generate`.

## DEBUG PATH
1. Request not routed → check `app/route/route.go` and feature `route_*.go`.
2. Request routed but wrong output → check `handler/*.go` then `logic/*.go`.
3. Infra/resource failures → check `repository/*.go` and startup init state.
4. gRPC auth failures → check `grpc/middleware/plugin_verify.go` + `logic/plugin_credential_logic.go`.
5. Auth not working → check `app/middleware/login_auth.go` + `app/grpc/client.go` (yggleaf).
