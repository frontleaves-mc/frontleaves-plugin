## CONVENTIONS
- Handler pattern: `grpcHandler` base struct (name, log, rdb) + domain service sub-structs (`essentialsService`, `statusService`, `titleService`).
- Each handler constructor (`New*Handler`) takes `(ctx, grpc.ServiceRegistrar)`, creates logic deps internally.
- Stream handlers: `ServerEventStream`, `EssentialsQueryStream` ‚Äî manage bidirectional streaming.
- Auth: metadata `plugin-secret-key` verified via `PluginCredentialLogic` in both unary and stream interceptors.
- Logs use `xLog.NamedGRPC` tag.
- `RegisterGRPCServices` also initializes `SchedulerEngine` (announcement cron) and calls `RecoverFromDatabase`.
- Tests: `handler/*_test.go` ‚Äî use `testify` assertions.

## ANTI-PATTERNS
- Do not edit files in `gen/` ‚Äî they are auto-generated by `buf generate`.
- Do not bypass `PluginVerify` interceptor; all plugin calls must authenticate.
- Do not create gRPC clients inside handlers; use context-injected deps from startup.
- Do not mix HTTP handler patterns with gRPC handler patterns.
